# Part A · 产品需求文档 (PRD)

---

## 1. 产品概述

### 1.1 产品名称与定位

**NovelFusion AI** — 多源小说智能分析与变体生成平台

> **一句话**：导入两部或多部小说，AI 深度解构叙事基因，对比融合，生成原创变体。

### 1.2 目标用户画像（User Personas）

#### Persona A · 网文创作者「林子豪」
- **年龄**：24 岁，全职写手
- **痛点**：卡文时缺乏灵感，参考大量作品但不知如何融合精华
- **目标**：分析爆款小说结构，生成自己的同类型变体
- **使用频率**：每周 3 ~ 5 次
- **技术熟悉度**：中等，不懂代码但善用 AI 工具

#### Persona B · 文学研究者「陈雅欣」
- **年龄**：31 岁，高校文学系讲师
- **痛点**：手动对比多部作品耗时极长，可视化呈现困难
- **目标**：快速获取多维度分析报告，用于学术研究和课堂教学
- **使用频率**：项目制，一次集中使用 2 ~ 3 天
- **技术熟悉度**：低，需要引导和帮助文档

#### Persona C · 出版编辑「Marcus T.」
- **年龄**：38 岁，资深编辑
- **痛点**：审稿效率低，难以快速判断作品风格定位
- **目标**：批量分析投稿作品，与畅销书对比，辅助选稿决策
- **使用频率**：每天使用
- **技术熟悉度**：高，熟悉各类 SaaS 工具，对导出报告格式有要求

### 1.3 核心用户旅程（完整版）

```
[首次访问]
    ↓
Onboarding 引导（3 步）
    ↓ 跳过 / 完成
[控制台首页 /dashboard]
    ↓
配置 LLM 模型 → 测试连接 → 保存配置
    ↓
新建会话 → 进入 /upload
    ↓
拖拽上传小说文件（≥ 2 部）→ 文件校验
    ↓
点击「开始清洗」→ 实时进度展示
    ↓
清洗结果预览 → 确认继续
    ↓
点击「开始分析」→ 并行 LLM 分析任务
    ↓
[分析详情页 /analysis/:sessionId]
    ↓
浏览 5 个分析维度 Tab → 交互图表探索
    ↓
跳转 [对比分析页 /compare]
    ↓
选择对比维度 → 查看兼容性评分 → 确定融合策略
    ↓
跳转 [变体生成页 /generate]
    ↓
配置生成参数 → 点击生成 → 实时流式输出
    ↓
编辑修改 → 导出文件
    ↓
[会话历史 /history] 查看所有历史记录
```

### 1.4 产品边界（Out of Scope）

- 不提供在线阅读版权小说（用户自行上传）
- 不支持实时协作多人编辑
- 不提供移动端原生 App（Web 响应式支持移动浏览器）
- 不内置小说发布或社区功能

#### 4.2.3 LLM 配置面板（完整规格）

**组件名**：`<LLMConfigPanel />`

**字段规格**：

| 字段 | 控件 | 校验规则 | 说明 |
|------|------|----------|------|
| 模型提供商 | Select | 必填 | OpenAI / Gemini / Anthropic / Ollama / 自定义 |
| API Endpoint | Text Input | URL 格式校验 | 自定义时必填，预设提供商自动填充 |
| API Key | Password Input | 非空 + 格式检测 | 输入时实时遮掩，可切换可见 |
| 模型版本 | Select + 刷新按钮 | 必填 | 连接成功后动态拉取，失败时显示手动输入框 |
| Max Tokens | Range Slider | 512 ~ 128000 | 右侧显示实时数值 |
| Temperature | Range Slider | 0.0 ~ 2.0，步长 0.1 | 下方显示含义标注（保守 → 创意） |
| Top-P | Range Slider | 0.0 ~ 1.0 | 与 Temperature 联动提示（两者均高时警告） |
| Frequency Penalty | Range Slider | 0.0 ~ 2.0 | 控制重复度，默认 0.3 |
| System Prompt 前缀 | Textarea | 最多 500 字 | 含字数计数器 |

**预设配置管理**：
- 最多保存 5 套预设（分析模式 / 创作模式 / 快速模式等）
- 每套预设支持：命名 / 保存 / 加载 / 删除 / 导入 / 导出（JSON）
- 预设列表用 Dropdown 切换，当前激活预设高亮显示

**连接状态机**：

```
idle ──点击「测试连接」──→ testing ──成功──→ connected
                                       └─失败──→ error
connected ──修改任意字段──→ dirty（黄色警告徽章）
error ──点击「查看详情」──→ 展开错误 tooltip（含错误码 + 排查建议）
```

**快捷操作区**：
- `⌘ + S`：保存当前配置
- `⌘ + Enter`：测试连接
- `⌘ + R`：重置为上次保存的配置

### 4.3 文本上传页 `/upload`

#### 4.3.1 文件上传区

**组件名**：`<NovelUploader />`

**支持格式与处理方式**：

| 格式 | 解析方式 | 注意事项 |
|------|----------|----------|
| `.txt` | 直接读取 | 自动检测编码（GBK / UTF-8）|
| `.epub` | epub.js 解析 | 保留章节结构元数据 |
| `.pdf` | pdf-parse | 扫描版可能失效，显示警告 |
| `.md` | remark 解析 | 保留标题层级作为章节边界 |
| `.docx` | mammoth | 保留粗体/斜体为强调标记 |

**交互规格**：
- 拖拽区域有虚线边框 + 图标，hover 时边框变为主色调实线
- 同时支持：点击选择 / 拖拽 / 粘贴文件路径
- 上传中：每个文件独立进度条（0% → 100%）
- 上传完成：文件卡片变为绿色勾选态
- **文件数量限制**：最少 2 个，最多 5 个（超出时自动提示并拒绝）
- **文件大小**：单文件 ≤ 50MB；超出时弹出警告但允许强制继续（提示将影响分析速度）

**文件卡片（FileCard）字段**：

```
┌─────────────────────────────────────────────┐
│ [📖图标] 书名输入框（可重命名）  [×删除]   │
│ 格式徽章  文件大小  预估字数               │
│ 状态：上传中 / 就绪 / 清洗中 / 清洗完成   │
│ ████████████░░ 75%   预计剩余 12s         │
└─────────────────────────────────────────────┘
```

#### 4.3.2 数据清洗模块（完整规格）

**清洗步骤与可视化**：

```
Step 1 ◉  编码检测与转换       → 显示：检测到 GBK，已转换为 UTF-8
Step 2 ◉  噪声内容移除         → 显示：移除广告 12 处 / 版权声明 2 处
Step 3 ◉  特殊字符标准化       → 显示：处理全角字符 340 个
Step 4 ◉  章节边界识别         → 显示：识别章节 88 个（置信度 94%）
Step 5 ◉  段落结构化           → 显示：结构化段落 2,340 段
Step 6 ◉  元数据提取           → 显示：书名 / 作者 / 字数 / 语言
```

**置信度不足处理**：
- 章节识别置信度 < 70% 时，显示黄色警告 + 「手动调整章节边界」入口
- 手动调整界面：左侧文本，右侧章节列表，可拖动分隔线重新划定章节

**清洗结果预览（Diff 视图）**：
- 并排对比：左侧原始文本（只读），右侧清洗后文本
- 删除内容：红色删除线；新增内容：绿色高亮
- 支持逐一「接受 / 拒绝」修改（精细控制模式）
- 底部统计栏：原始字数 → 清洗后字数 · 减少 X% · 章节数 · 对话占比%

**数据结构输出（JSON Schema）**：

```json
{

  "author": "作者（若可识别）",

  "cleanedAt": "ISO8601 timestamp",

    {
      "chapterIndex": 1,

      "content": "正文内容...",
      "wordCount": 3200,
      "paragraphs": ["段落1", "段落2"],
      "dialogueLines": ["「对话内容」"],

#### 4.4.2 Tab 1：章节结构分析

**分析输出**：
- 章节总数 / 平均字数 / 最长章节 / 最短章节（KPI 卡片组）
- 叙事三段分布：前 1/3 / 中 1/3 / 后 1/3 字数比例（面积图）
- 章节标题风格分类：悬念型 / 描述型 / 隐喻型 / 人名型（饼图 + 例子）
- 高潮章节识别：情感强度热力图（x 轴=章节，y 轴=情感强度）

**图表组件**：

| 组件名 | 类型 | 交互 |
|--------|------|------|
| `<ChapterLengthBarChart />` | 柱状图 | hover 显示章节标题 tooltip |
| `<NarrativePacingLine />` | 折线图 | 点击折线节点跳转章节预览 |
| `<ChapterHeatmap />` | 热力图 | 颜色渐变：冷色（平静）→ 暖色（高潮）|

#### 4.4.3 Tab 2：情节分析

**分析输出**：
- 主线剧情摘要（AI 生成，3 ~ 5 句话，可展开完整版）
- 情节结构类型识别（英雄之旅 / 三幕式 / 起承转合 / 非线性）附置信度
- 核心冲突矩阵（人 vs 人 · 人 vs 自然 · 人 vs 社会 · 人 vs 自我）
- 情节转折点列表：章节号 · 转折描述 · 影响评分（1~10）
- 悬念密度曲线（逐章节悬念指数折线图）
- 伏笔 → 揭示 对应关系表（左列：伏笔，右列：揭示位置）

**图表组件**：

| 组件名 | 类型 | 交互 |
|--------|------|------|
| `<PlotArcTimeline />` | 横向时间轴 | 拖动缩放，点击节点展开详情 |
| `<ConflictTypeRadar />` | 雷达图 | 多书叠加对比模式 |
| `<TurningPointCards />` | 卡片列表 | 按影响评分排序/过滤 |
| `<ForeshadowingTable />` | 对照表 | 高亮行、跳转章节 |

#### 4.4.4 Tab 3：写作特效与手法

**分析输出**：
- 修辞手法频次统计（比喻 · 拟人 · 排比 · 对比 · 反问 · 夸张 · 通感）
- 叙事视角鉴定（第一人称 / 第三有限 / 全知 / 混合）+ 置信度 + 例句
- 对话 vs 描述占比（环形图，可按章节段落筛选）
- 时间叙事手法（顺叙 / 倒叙 / 插叙 / 预叙）占比统计
- 场景切换频率曲线（每 N 章切换次数）
- 情感渲染技巧标签云（慢镜头 · 白描 · 意识流 · 蒙太奇 · 象征）

**图表组件**：

| 组件名 | 类型 | 交互 |
|--------|------|------|
| `<RhetoricBarChart />` | 水平条形图 | 点击展开该修辞手法例句 |

## 6. 错误与边界状态

### 6.1 错误类型与处理策略

| 错误类型 | 触发场景 | 处理方式 |
|----------|----------|----------|
| 网络断开 | API 请求超时 / 无网 | 顶部横幅提示 + 任务暂停 + 自动重连 |
| LLM 认证失败 | API Key 无效 | 内联错误 + 跳转配置面板 |
| Token 超限 | 文章过长超出模型限制 | 警告 + 提供「分块处理」选项 |
| 文件格式不支持 | 上传 .zip / .exe 等 | 即时 Toast 错误 + 列出支持格式 |
| 文件解析失败 | 损坏 PDF / 乱码 TXT | 卡片显示错误 + 「尝试强制解析」选项 |
| 分析任务失败 | LLM 返回空/异常 | 单维度错误态 + 「重试此维度」按钮 |
| 浏览器存储满 | IndexedDB 配额耗尽 | 提示清理旧数据 + 引导导出备份 |

### 6.2 空状态（Empty State）设计规格

每个可能出现空态的区域均需定义：

| 页面/区域 | 空态文案 | 操作引导 |
|-----------|----------|----------|
| 历史记录列表 | 「还没有分析记录」| [新建会话] 按钮 |
| 分析结果 Tab | 「该维度分析进行中，请稍候」| 进度动画 |
| 人物关系图谱 | 「未检测到明确人物关系」| 提示调整 Token 或重新分析 |
| 世界观信仰体系 | 「本书未涉及宗教/信仰设定」| 无操作，仅说明 |
| 搜索结果 | 「没有找到相关内容」| 清除搜索词建议 |

### 6.3 降级策略

- LLM 不可用时：提供「演示模式」（使用预置 Mock 数据展示所有功能）
- 图表库加载失败时：降级为文字表格展示
- IndexedDB 不可用时：降级为 sessionStorage（提示数据不持久）

## 7. 安全与隐私

### 7.1 API Key 安全

- API Key 仅存储在客户端（localStorage 加密存储，建议用 AES-256 + 用户定义密码）
- 绝不将 API Key 发送至本产品后端（若有后端代理，需走服务端 Secrets 管理）
- 页面离开/刷新时提示「API Key 已仅保存在本地，不会上传云端」
- 提供「会话结束后清除 Key」选项

### 7.2 用户数据

- 所有上传文本仅在浏览器本地处理或通过用户配置的 LLM Endpoint 发送
- 不向任何第三方服务器发送用户小说内容（除用户自选的 LLM 服务）
- 首次使用前显示数据使用说明（Privacy Notice 弹窗）

## 11. 设计原则

### 原则一：透明可信（Transparent）
> AI 处理的每个步骤都对用户可见，没有黑盒。

- 所有后台任务显示进度 + 步骤说明
- LLM 的推理过程（Chain-of-Thought）可选可见
- 分析结论附置信度百分比
- 错误信息提供具体原因，而非「操作失败」

### 原则二：渐进披露（Progressive Disclosure）
> 先展示最重要的信息，细节按需展开。

- 分析结果先显示摘要卡片，详细数据折叠在后
- 配置项按「基础 / 高级」分组，高级项默认折叠
- 图表先显示概览，点击/hover 再显示数据细节

### 原则三：流程导向（Flow-Oriented）
> 工具的核心价值是完成一次完整的分析 → 生成工作流。

- 每个页面底部有明确的「下一步」行动按钮
- 当前工作流位置始终在导航中可见（面包屑 + 进度指示）
- 未完成的步骤不会被跳过，但会提供明确提示

### 原则四：容错宽松（Forgiving）
> 任何操作都可以撤销、重试或修改。

- 文件上传后可随时删除或替换
- 清洗结果可逐项接受/拒绝
- 生成过程可暂停、取消并保留已生成内容
- 关键操作（删除 Session）需二次确认

### 原则五：专注高效（Focused）
> 减少非必要的视觉噪声，让用户专注于内容。

- 分析内容页不显示无关推广或功能入口
- 正在生成内容时，界面进入「专注模式」（弱化非核心 UI 元素）
- 快捷键覆盖所有核心操作，减少鼠标移动距离

## 16. 信息反馈体系

### 16.1 反馈层级

```
层级 1 · 即时反馈（< 100ms）
  → 按钮 hover / active 状态变化
  → 输入框 focus 边框变化
  → Checkbox / Radio 点击切换

层级 2 · 短时反馈（1 ~ 5s）
  → Toast 通知（操作成功/失败）
  → Tooltip 提示

层级 3 · 持续反馈（任务进行期间）
  → 进度条 + 百分比
  → Step Progress（步骤完成状态）
  → 流式输出光标

层级 4 · 结果反馈（任务完成后）
  → 空态插画 + 说明文字
  → 错误状态 + 恢复操作
  → 完成状态 + 下一步引导
```

### 16.2 空态（Empty State）视觉规范

每个空态包含以下三要素：

```
[SVG 插画] ← 轻量线性风格，主色调，60px × 60px 以内
    ↓
主要说明文字（--text-xl，--font-semibold）
    ↓
次要说明文字（--text-sm，--text-secondary，最多 2 行）
    ↓
[行动按钮]（Primary 或 Ghost 变体）
```

**插画风格**：统一使用线性（Line Art）风格，主色 + 描边，避免复杂彩色插画影响性能和可维护性。

### 16.3 错误状态设计

**内联错误**（表单字段下方）：
```
[⚠ 图标] 错误说明文字（--text-sm，--color-error）
[💡 图标] 修复建议（可选，--text-sm，--text-secondary）
```

**区块错误**（分析维度失败）：
```
┌────────────────────────────────────────┐
│ ✕  人物关系分析失败                   │

**全页错误**（会话数据丢失等）：
- 大尺寸错误插画 + 清晰说明 + 「返回首页」+ 「联系支持」

## 17. 无障碍规范（WCAG 2.1 AA）

### 17.1 颜色对比度

| 文字类型 | 最低对比度 |
|----------|-----------|
| 正文（≥18px）| 3:1 |
| 正文（<18px）| 4.5:1 |
| 大文字（≥24px 或 18px Bold）| 3:1 |
| 图形元素（图标、图表轴线）| 3:1 |

**已验证**：
- `--text-primary` (#E6EDF3) on `--bg-elevated` (#161B22)：对比度 ≈ 13.5:1 ✓
- `--text-secondary` (#8B949E) on `--bg-elevated` (#161B22)：对比度 ≈ 5.2:1 ✓
- `--color-purple-400` (#8B6FFF) on `--bg-elevated` (#161B22)：对比度 ≈ 5.8:1 ✓

### 17.2 键盘导航规范

- 所有交互元素均可通过 `Tab` / `Shift+Tab` 聚焦
- 聚焦时显示明显的 `focus-visible` 样式（2px 品牌色外轮廓 + 2px offset）
- 模态框开启时，焦点被陷阱在 Modal 内（Focus Trap）
- Dropdown/Menu 支持方向键导航，`Enter` 确认，`Esc` 关闭
- 图表提供文字替代版本（表格视图切换）

```css
/* 全局 Focus 样式 */
:focus-visible {
  outline: 2px solid var(--color-purple-400);
  outline-offset: 2px;
  border-radius: var(--radius-sm);
}
```

### 17.3 ARIA 标签规范

| 组件 | 必须属性 |
|------|---------|
| 图标按钮 | `aria-label="操作说明"` |
| 进度条 | `role="progressbar" aria-valuenow aria-valuemin aria-valuemax` |
| 模态框 | `role="dialog" aria-modal="true" aria-labelledby` |
| Tab 组 | `role="tablist" / tab / tabpanel"` |
| 状态徽章 | `aria-live="polite"` |
| 错误提示 | `role="alert" aria-live="assertive"` |
| 加载状态 | `aria-busy="true" aria-label="正在加载"` |

### 17.4 动效无障碍

遵守 CSS `prefers-reduced-motion`：
```css
@media (prefers-reduced-motion: reduce) {
  * {
    animation-duration: 0.01ms !important;
    transition-duration: 0.01ms !important;
  }
}
```

## 18. 用户引导设计

### 18.1 Onboarding 流程详细规格

**Step 1 · 欢迎**
- 中央卡片，品牌渐变背景
- 大号标题 + 一句话定位
- 3 个功能亮点（图标 + 2 行文字，水平排列）
- 按钮：「开始配置」(Primary) + 「演示模式体验」(Ghost)

**Step 2 · 配置 LLM**
- 嵌入精简版 `<LLMConfigPanel />`（只显示必填 4 项）
- 提供「推荐选项」快速填充（适合不熟悉的用户）
- 底部显示：「你的 API Key 只存储在本地，不会上传至任何服务器」
- 按钮：「测试并继续」(Primary) + 「跳过」(Ghost)

**Step 3 · 准备好了**
- 成功状态插画（动态绘制效果）
- 快捷键速查卡（5 个最常用快捷键）
- 按钮：「开始使用 →」(Primary Large)

### 18.2 Tooltip 帮助系统

**触发方式**：每个配置项标签后跟 `ⓘ` 图标，hover 显示 Tooltip

**Tooltip 内容规范**：
- 用途说明（1 句话）
- 推荐值 / 取值范围（若有）
- 注意事项（若有，用黄色 ⚠ 前缀）

**示例**：
```
Temperature
ⓘ ─────────────────────────────────
  控制输出的随机性和创意程度
  推荐值：分析任务 0.3，创作任务 0.8
  ⚠ 与 Top-P 同时调高时效果叠加，谨慎使用
────────────────────────────────────
```

### 18.3 上下文提示（Contextual Hints）

在关键操作前，以「信息条（Info Banner）」形式提示：

| 位置 | 提示内容 |
|------|----------|
| 上传页（上传 1 个文件时）| 「还需要至少 1 部小说才能进行对比分析」|
| 清洗预览页 | 「建议检查高亮的修改项，特别是章节边界的调整」|
| 分析页（分析中）| 「分析完成前可以浏览已完成的维度」|
| 生成页（创新程度 > 8）| 「高创新程度可能导致与原著差异较大，请确认是否符合预期」|
