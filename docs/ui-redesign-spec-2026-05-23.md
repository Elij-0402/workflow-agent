# NovelFusion UI 重构说明

日期：2026-05-23

## 目标

本轮重构把产品从“按功能分散跳转”改成“按项目推进”的工作流界面。重点不是新增能力，而是让现有能力拥有更清楚的入口、层级和状态表达。

## 设计原则

1. 项目优先：分析、蓝图、简报、结果都回到项目主线下组织。
2. 中文优先：默认文案使用简体中文，减少中英混杂和工程感过重的标签。
3. SaaS 化：用稳定导航、明确状态和统一的卡片/列表结构替代工具拼盘感。
4. 双书优先：双书项目是主路线，单书流程保留但降为兼容模式。

## 信息架构

- 项目：项目总览页，显示阶段、下一步、关键指标和最近动态。
- 创作台：管理创意简报，承接 outline 预览与后续迭代。
- 对比：从双书项目进入分析图谱，对参考书做横向观察。
- 资料库：归档项目与历史结果的安置区。
- 设置：模型配置与系统级设置。

## 关键改动

### 1. 全局外壳

- 左侧导航统一为 项目 / 创作台 / 对比 / 资料库 / 设置。
- 顶栏加入工作台语义和搜索占位，视觉更接近产品后台而非实验页面。
- 主色从偏金色切到偏靛蓝系，保留深色底与清晰层级。

### 2. 项目首页

- 新增项目 KPI：活跃项目、待确认蓝图、活跃简报、已生成版本。
- 项目卡直接展示阶段、下一步和最近状态。
- 右侧增加最近动态和页面规则，帮助用户快速分配注意力。

### 3. 项目总览

- 模块导航改为：概览 / 参考书 / 分析图谱 / 蓝图工作台 / 创意简报 / 结果实验室。
- 概览页强调“下一步”与“最近状态”，不再只是静态说明。
- 参考书区和项目状态区拆开，形成更明确的上下文与决策区。

### 4. 创作台

- 创作台成为真实入口页，不再重定向回项目页。
- 支持查看可用简报、按项目聚焦、从侧边 Sheet 新建简报。
- `/studio/new` 改为真正的简报创建页。

### 5. 对比页

- 对比页改为真实项目入口，支持选择一个或多个双书项目展开图谱。
- 不再把 compare 当作简单跳转器。

### 6. 资料库

- 归档项目迁移到独立资料库页面，减少主列表噪音。

## 暂未覆盖

- 设置页仍沿用现有结构，仅继承新的全局壳层与配色。
- 工作台内部的三栏强化、简报/结果更深层联动还可以继续细化。

## 后续建议

1. 为工作台补一层固定三栏布局：左侧参考书，中间蓝图/分析，右侧简报与结果摘要。
2. 给结果实验室加入版本筛选、简报来源标识和最近生成时间线。
3. 为项目总览补充“最近生成结果”卡片，直接连接到对应版本。
